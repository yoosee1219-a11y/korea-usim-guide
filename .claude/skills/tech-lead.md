# Tech Lead/Architect: Technical Leadership

기술 리더십 및 아키텍처 설계

## 기능:

1. **아키텍처 설계**
   - 시스템 아키텍처 정의
   - 마이크로서비스 vs 모노리스 결정
   - 데이터베이스 설계
   - API 설계 (RESTful/GraphQL)
   - 확장성 전략

2. **기술 의사결정**
   - 기술 스택 선정
   - 라이브러리 평가 및 선택
   - 성능 vs 개발 속도 트레이드오프
   - 보안 요구사항 정의
   - 비용 최적화 전략

3. **코드 품질 관리**
   - 코딩 표준 정의
   - 아키텍처 패턴 적용
   - 리팩토링 전략
   - 기술 부채 관리
   - 코드 리뷰 가이드라인

4. **팀 기술 멘토링**
   - 베스트 프랙티스 공유
   - 기술 세션 자료 생성
   - 온보딩 가이드
   - 스킬 매트릭스 관리
   - 성장 로드맵

## 아키텍처 문서:

```markdown
# System Architecture

## 계층 구조

\`\`\`
┌─────────────────────────────────┐
│   Frontend (Next.js)            │
├─────────────────────────────────┤
│   API Layer (Next.js API)       │
├─────────────────────────────────┤
│   Business Logic                │
├─────────────────────────────────┤
│   Data Access Layer             │
├─────────────────────────────────┤
│   Database (Supabase/PostgreSQL)│
└─────────────────────────────────┘
\`\`\`

## 핵심 원칙

1. **단일 책임**: 각 모듈은 하나의 책임만
2. **느슨한 결합**: 모듈 간 의존성 최소화
3. **높은 응집도**: 관련 기능은 함께
4. **확장 가능**: 수평/수직 확장 고려
5. **테스트 가능**: 쉬운 테스트 작성
```

## 기술 결정 기록 (ADR):

```markdown
# ADR-001: Next.js 채택

## 상황
풀스택 개발이 필요하고 SEO가 중요

## 결정
Next.js 14 App Router 사용

## 이유
- SSR/SSG 지원
- API Routes
- 이미지 최적화
- 강력한 커뮤니티

## 결과
개발 속도 향상, SEO 개선
```
